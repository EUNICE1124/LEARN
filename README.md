# LEARN
GitHub Learning & Practice Schedule (1 Week)

Day 1 – Introduction & Setup

Objective: Understand GitHub basics and set up accounts.

Tasks:

Each team member creates a GitHub account (if not done).

Install Git on your computer and configure:

Day 2 – Git Basics

Objective: Learn local Git commands.

Tasks:

Understand git init, git add, git commit, git status.

Practice creating a local repository and making commits.

Team Exercise: Each member creates a simple .txt file in their local repo and commits it.

Day 3 – Cloning & Branching

Objective: Work on team branches.

Tasks:

Learn git clone to copy the team repository locally.

Learn to create and switch branches:

git checkout -b feature/your-feature

Team Exercise: Each member creates a branch for a feature, makes a small change, commits, and pushes to GitHub.

Day 4 – Pull Requests & Code Review

Objective: Collaborate using Pull Requests.

Tasks:

Learn how to create a Pull Request (PR) on GitHub.

Learn how to review and comment on PRs.

Team Exercise: Each member submits a PR for their feature branch. Other members review and approve it.

Day 5 – GitHub Issues & Project Boards

Objective: Organize team tasks efficiently.

Tasks:

Learn to create Issues for bugs, tasks, and features.

Explore Projects/Boards (Kanban style).

Team Exercise: Create a board for your project and assign tasks to members.

Day 6 – Integration & Team Workflow

Objective: Consolidate all skills in a mini project.

Tasks:

Each member works on a small feature in their branch.
Submit PRs for review, merge into main.
Update Issues and mark tasks as completed.
Team Reflection: Discuss what worked, what was challenging, and workflow improvements.
